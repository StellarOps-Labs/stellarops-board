# Contributor Workflows

Clear issues help contributors spend time building instead of guessing.

## Choosing Work

Contributors can scan by difficulty, label, readiness status, repository, and last update. Beginner issues should include especially clear files to update and acceptance criteria.

## Completing Work

A strong issue explains expected behavior and review criteria, so contributors can check their own work before opening a pull request.

## Maintainer Benefits

When contributors understand scope, maintainers spend less time clarifying basics and more time reviewing useful changes.
